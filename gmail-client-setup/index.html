Gmail SMTP Setup Guide

Configure your Gmail account for sending emails

Prerequisites

  • A Gmail account
  • 2-Factor Authentication enabled

Setup Steps

1Generate Gmail App Password

  1. Go to Google Account Settings:
    Visit: https://myaccount.google.com/apppasswords
    Sign in with your Gmail account
  2. Enable 2-Factor Authentication (if not already enabled):
    Go to: https://myaccount.google.com/security
    Click "2-Step Verification" and follow the setup wizard
  3. Create App Password:
    • Go back to: https://myaccount.google.com/apppasswords
    • Select app: "Mail"
    • Select device: "Other (Custom name)"
    • Enter a descriptive name (e.g., "Email Client")
    • Click "Generate"
    • Copy the 16-character password (looks like: abcd efgh ijkl mnop)
    • Remove spaces when storing: abcdefghijklmnop

2Configure Your Email Client

Use these SMTP settings in your email client:

Setting Value
SMTP Server smtp.gmail.com
Port 587 (with STARTTLS)
Username Your full Gmail address (e.g., linux.trevor@gmail.com)
Password The 16-character app password (no spaces)
Security STARTTLS or TLS
⚠️ Important: Use the app password, not your regular Gmail password!

3Test Your Configuration

Send a test email from your email client to verify the setup works correctly.

If configured properly, you should be able to send emails using your Gmail account through any SMTP-compatible email client or application.

Gmail SMTP Limits

Feature Limit
Sending limit 500 emails per day (free accounts)
Recipients per message 500 recipients
Message size 25 MB (including attachments)
Google Workspace 2,000 emails per day (paid accounts)

Troubleshooting

"Username and Password not accepted"

  • Make sure 2FA is enabled on the Gmail account
  • App password must be 16 characters, no spaces
  • Try regenerating the app password

"SMTP AUTH extension not supported"

  • Make sure you're using port 587 (not 465)
  • Enable "Less secure app access" if app passwords don't work
  • Check that SMTP is enabled in Gmail settings

"Daily sending quota exceeded"

  • Gmail free accounts: 500 emails/day
  • Wait 24 hours or upgrade to Google Workspace

Security Best Practices

  • Use App Passwords - Never use your main Gmail password for SMTP
  • Enable 2-Factor Authentication - Required for app passwords
  • Rotate passwords - Regenerate app passwords periodically
  • Monitor usage - Check Gmail "Sent" folder for unexpected activity
  • Revoke unused passwords - Remove app passwords you're no longer using
