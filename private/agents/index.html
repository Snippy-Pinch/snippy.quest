- Task needs isolation from main session history
- You want a different model or thinking level for the task
- One-shot reminders ("remind me in 20 minutes")
- Output should deliver directly to a channel without main session involvement

**Tip:** Batch similar periodic checks into `HEARTBEAT.md` instead of creating multiple cron jobs. Use cron for precise schedules and standalone tasks.

## Image Generation

Generate images using HuggingFace (with automatic token rotation and Pollinations.ai fallback).

```bash
# Generate an image
python3 scripts/generate_image_smart.py "your prompt here" -o data/output.png
```

### CRITICAL: Sending the image

After generating an image, you MUST include a `MEDIA:` line in your response text so the gateway sends it as an attachment. Example:

```
Here's the image you requested!
MEDIA:data/output.png
```

The `MEDIA:` token must be on its own line, NOT inside a code block. Use a unique filename for each image (e.g. include a timestamp).

### Full options

```bash
# With negative prompt
python3 scripts/generate_image_smart.py "prompt" --negative "bad quality, blurry" -o data/output.png

# Force Pollinations fallback (free, no token needed)
python3 scripts/generate_image_smart.py "prompt" --pollinations-only -o data/output.png
```

## Database

You have a SQLite database at `data/snippy.db`. Use it to store anything the group needs: scores, bets, lists, facts, whatever.

### Usage

```bash
# List all tables
python3 scripts/db.py tables

# Show a table's schema
python3 scripts/db.py schema 

# Run a read query (returns formatted table with headers)
python3 scripts/db.py query "SELECT * FROM scores ORDER BY points DESC"

# Run a write statement (CREATE TABLE, INSERT, UPDATE, DELETE)
python3 scripts/db.py exec "CREATE TABLE scores (name TEXT PRIMARY KEY, points INTEGER DEFAULT 0)"
python3 scripts/db.py exec "INSERT INTO scores (name, points) VALUES ('Dave', 10)"
```

### Tips
- Create whatever tables you need on the fly ‚Äî no need to ask permission
- Use `query` for SELECT statements, `exec` for everything else
- The database persists across conversations
- Always quote the SQL argument so the shell doesn't mangle it
